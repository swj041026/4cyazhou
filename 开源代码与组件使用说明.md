# 开源代码与组件使用说明

## 项目概述

**项目名称**：崖城风骨——崖州古城建筑文化数字图鉴  
**项目类型**：全国大学生计算机设计大赛参赛作品（信息可视化·交互信息设计赛道）  
**技术栈**：原生 HTML5 + CSS3 + JavaScript (ES6+)，Node.js 后端服务  
**开源许可证**：MIT License

---

## 一、项目依赖分析

### 1.1 运行时依赖

本项目**未使用任何 npm 第三方依赖包**。package.json 中 `dependencies` 字段为空。

```json
{
  "name": "yazhou-guocai",
  "version": "1.0.0",
  "license": "MIT",
  "engines": {
    "node": ">=14.0.0"
  }
}
```

### 1.2 开发/构建依赖

本项目**无构建工具链**，不使用 Webpack、Vite、Rollup 等打包工具，也不使用 Babel、TypeScript 等转译工具。

## 二、核心技术栈（原生实现）

### 2.1 前端技术

| 技术 | 用途 | 说明 |
|------|------|------|
| HTML5 | 页面结构 | 语义化标签，无障碍支持 |

| JavaScript (ES6+) | 交互逻辑 | 原生 API，无框架依赖 |

### 2.2 后端技术

| 技术 | 用途 | 说明 |
|------|------|------|
| Node.js | 运行时环境 | 原生 http/https 模块 |
| 原生模块 | 文件服务 | fs, path, url 等内置模块 |

### 2.3 外部服务

| 服务 | 用途 | 说明 |
|------|------|------|
| DeepSeek API | AI 对话服务 | 智能助手功能，需配置 API Key |

## 三、开源组件使用情况

### 3.1 确认使用的开源资源

#### 1. Google Fonts（外部字体服务）
- **资源地址**：https://fonts.googleapis.com
- **使用字体**：
  - Noto Serif SC（思源宋体）- 标题、正文
  - Noto Sans SC（思源黑体）- 界面文字
- **许可证**：SIL Open Font License 1.1
- **使用位置**：index.html 及所有页面 HTML 文件

```html
<link href="https://fonts.googleapis.com/css2?family=Noto+Serif+SC:wght@400;600;700&family=Noto+Sans+SC:wght@300;400;500;700&display=swap" rel="stylesheet">
```

#### 2. Node.js 运行时
- **版本要求**：>= 22.14.0
- **许可证**：MIT License
- **官网**：https://nodejs.org/

#### 3. Docker（部署环境）
- **基础镜像**：node:18-alpine
- **许可证**：Apache License 2.0
- **使用位置**：Dockerfile

## 四、自定义实现的核心功能

以下功能均为**项目团队自主开发**，未使用第三方库：

### 4.1 前端自定义模块

| 模块 | 文件路径 | 功能描述 |
|------|----------|----------|
| 工具函数 | js/global.js | DOM 选择器、事件绑定、节流函数、本地存储封装 |
| 导航栏 | js/global.js | 滚动响应、移动端菜单切换 |
| 页面进度条 | js/global.js | 滚动进度计算与可视化 |
| 全局控件 | js/global.js | 背景音乐控制、回到顶部 |
| 弹窗组件 | js/global.js | 模态框显示/隐藏、内容注入 |
| 引导蒙层 | js/global.js | 首次访问引导、本地存储标记 |
| 滚动动画 | js/global.js | IntersectionObserver 实现元素进入视口动画 |
| AI 智能助手 | js/global.js | DeepSeek API 集成、对话历史管理、拖拽功能 |
| 城门转盘 | js/wheel.js | 交互式城门信息展示组件 |
| 官员转盘 | js/officials-wheel.js | 历史人物信息轮播组件 |

## 六、部署与运行

### 6.1 本地开发

```bash
# 安装依赖（无第三方依赖，此步骤可选）
npm install

# 启动开发服务器
npm run dev
# 或
node api-server.js
```

### 6.2 Docker 部署

```bash
# 构建镜像
docker build -t yazhou-guocai .

# 运行容器
docker run -p 6677:8080 -e DEEPSEEK_API_KEY=your_key yazhou-guocai
```

### 6.3 Docker Compose 部署

```bash
# 配置环境变量后启动
docker-compose up -d
```

## 八、总结

本项目是一个**纯原生技术栈**实现的 Web 应用，具有以下特点：

1. **零第三方依赖**：前端不使用任何 npm 包，所有功能自主实现
2. **轻量级架构**：无构建步骤，直接部署 HTML/CSS/JS 文件
3. **现代原生 API**：充分使用 ES6+ 特性、CSS Variables、IntersectionObserver 等
4. **模块化设计**：代码按功能模块组织，便于维护
5. **AI 集成**：通过 DeepSeek API 提供智能对话功能

如需进一步了解具体实现细节，请查阅项目源代码中的注释说明。
